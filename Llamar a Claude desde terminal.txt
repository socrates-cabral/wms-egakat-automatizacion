  El corazón de Claude Code: la función query() que orquesta conversación, herramientas y gestión de contexto en un loop infinito hasta que decide parar.   

  ---
  Patrones extraíbles para tus proyectos

  1. Token budget con diminishing returns — aplicable a multi_llm_analyst.py

  El spec revela la lógica exacta de cuándo parar un agente:

  # Patrón spec/01 — Token budget decision
  COMPLETION_THRESHOLD = 0.90   # parar si >90% del budget usado
  DIMINISHING_THRESHOLD = 500   # tokens delta "pequeño" = está dando vueltas
  MIN_CONTINUATIONS = 3         # esperar 3 iteraciones antes de detectar plateau

  def check_token_budget(continuation_count, last_delta, current_tokens, budget):
      pct = round(current_tokens / budget * 100)
      is_diminishing = (
          continuation_count >= 3 and
          last_delta < DIMINISHING_THRESHOLD
      )
      if not is_diminishing and current_tokens < budget * COMPLETION_THRESHOLD:
          return "continue", f"Continuando... {pct}% del budget usado"
      return "stop", pct

  Hoy multi_llm_analyst.py no tiene ningún control de esto — si un LLM entra en loop, no hay freno.

  2. Tres niveles de compresión de contexto — patrón para AI_Agent

  ┌──────────────────┬─────────────────────────┬──────────────────────────────────────────────────────┐
  │      Nivel       │         Cuándo          │                     Qué elimina                      │
  ├──────────────────┼─────────────────────────┼──────────────────────────────────────────────────────┤
  │ Microcompact     │ Cada iteración          │ Mensajes meta intermedios, tool results descartables │
  ├──────────────────┼─────────────────────────┼──────────────────────────────────────────────────────┤
  │ Autocompact      │ Al 80% del contexto     │ Turns más antiguos, preserva últimos ~10             │
  ├──────────────────┼─────────────────────────┼──────────────────────────────────────────────────────┤
  │ Reactive compact │ Error "prompt too long" │ Compresión agresiva al 50%, una sola vez             │
  └──────────────────┴─────────────────────────┴──────────────────────────────────────────────────────┘

  ask_ai.py hoy construye el prompt completo cada vez. Si el historial crece, eventualmente va a explotar con un 413. El patrón de autocompact soluciona    
  esto.

  3. Stop reasons tipados — aplicable a run_agent.py del agente apuestas

  # En vez de solo True/False al terminar un agente, retornar razón tipada
  STOP_REASONS = {
      'end_turn',        # terminó naturalmente (sin tool_use)
      'max_turns',       # excedió límite de iteraciones
      'blocking_limit',  # contexto demasiado largo incluso tras compresión
      'budget_exhausted' # se acabaron los tokens del budget
  }

  Permite al orquestador reaccionar distinto según la razón.

  4. projectRoot inmutable vs cwd dinámico — aplica a ask_ai.py

  El spec es explícito: projectRoot se setea al arrancar y nunca cambia aunque el agente navegue entre carpetas. Es la identidad de la sesión. cwd sí       
  cambia. Esta distinción evita bugs donde un agente modifica archivos fuera de su scope esperado.

  5. Tombstone para mensajes huérfanos — patrón para streaming

  Si un stream cae a mitad, en vez de dejar el mensaje a medias en el historial, se marca como tombstone. Si en el futuro agregas streaming al agente de    
  apuestas o finanzas, este patrón evita que mensajes incompletos contaminen el historial.
